# CI/CD 實戰學習計畫

> 總時長：約 3 小時（含休息）
> 對象：有基本 Docker 經驗的初學者
> 前置條件：已完成 Docker 教育訓練

---

## 課程目標

- 理解 CI/CD 的概念與為什麼需要它
- 認識 GitHub Actions 的運作方式（Workflow / Job / Step / Action）
- 能撰寫基本與多 Job 的 workflow 檔案
- 實作兩個完整部署流程：
  - **Demo A**：純 HTML 靜態網站 → GitHub Pages（無需 Docker）
  - **Demo B**：Node.js + Redis + Docker Image → Docker Hub → Railway 雲端部署

## 兩個 Demo 對照

| 面向 | Demo A：GitHub Pages | Demo B：Railway |
|------|---------------------|-----------------|
| 專案位置 | [demo-01-github-pages/](demo-01-github-pages/) | [demo-02-railway-docker/](demo-02-railway-docker/) |
| 技術棧 | 純 HTML | Node.js + Redis |
| 容器化 | 不需要 | Dockerfile（單階段）+ docker-compose（本機含 Redis） |
| Job 數 | 1 個（deploy） | 3 個（test → build-push → deploy） |
| 需要的 Secrets | 0 個（用 `GITHUB_TOKEN`） | 4 個（Docker Hub 帳密 + Railway Token + Service ID） |
| 部署目標 | GitHub Pages `gh-pages` branch | Railway（拉 Docker Hub image） |
| 成品網址 | `{帳號}.github.io/demo-01-github-pages/` | `your-app.railway.app` |
| 適合場景 | 靜態頁面、文件、部落格 | 有後端邏輯 + 持久化狀態的 Web App |

## 學員需要準備

- GitHub 帳號（已安裝 Git）
- Docker Hub 帳號（並能產生 Access Token）
- Railway 帳號（用 GitHub 登入即可）
- 已完成 Docker 基礎教育訓練

## 延伸學習

| 主題 | 說明 |
|------|------|
| GitLab CI / Jenkins | 其他常見 CI/CD 平台，概念相通、語法不同 |
| ArgoCD | GitOps 風格的 CD 工具（搭配 Kubernetes） |
| GitHub Actions Marketplace | 社群提供的現成 Action |
| Matrix / Cache / Artifacts | GitHub Actions 進階功能 |
| Branch Protection Rule | 在 merge 前強制要求 CI 通過 |
| 其他部署目標 | AWS ECS、GCP Cloud Run、Vercel、Fly.io 等 |
